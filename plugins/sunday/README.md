# Sunday - Jira 기반 자동화 개발 워크플로우

Jira 티켓을 입력받아 요구사항 분석 → 코드 구현 → 코드 리뷰 → 요구사항 리뷰 → PR 오픈까지 **완전 자동화**하는 **Frontend 전용** 플러그인입니다.

## 사전 준비

```
/sunday:setup
```

Jira CLI 설치 및 로그인 설정을 진행합니다.

## 사용법

```bash
# 기본 사용 (develop의 가이드 참조)
/sunday:start JIRA-123

# 다른 브랜치의 가이드 참조
/sunday:start JIRA-123 --guide-branch main
```

지정된 Jira 티켓에 대해 다음 워크플로우를 **완전 자동화**로 수행합니다:

1. Jira 티켓 조회 & 품질 체크
2. 요구사항 분석 & 작업 계획 수립
3. 코드 구현 (apps/front 대상)
4. 코드 리뷰 (80점 통과까지 반복)
5. 요구사항 리뷰 (80점 통과까지 반복)
6. PR 생성 & Jira 코멘트 추가

## 핵심 설계 원칙

| 원칙 | 설명 |
|------|------|
| Frontend 전용 | `apps/front` 디렉토리 대상, 해당 하위 파일 변경사항만 체크 |
| 완전 자동화 | 한 번 호출하면 PR 오픈까지 사람 개입 없이 진행 |
| Jira CLI 사용 | MCP 대신 `jira` CLI 도구로 티켓 조회/업데이트 |
| frontend-doc 기준 | `apps/front/.claude/skills/frontend-doc`에 명시된 규칙만 감점 대상 |
| 가이드 브랜치 | `--guide-branch`로 다른 브랜치의 최신 가이드 참조 (기본값: `develop`) |
| 80점 게이트 | 리뷰 통과/재시도 결정, 통과할 때까지 무한 루프 |

## 컴포넌트

| 컴포넌트 | 설명 |
|----------|------|
| `setup` | Jira CLI 환경 설정 가이드 |
| `start` | 메인 워크플로우 진입점 |
| `planner` agent | 작업 계획 수립 |
| `code-reviewer` agent | 순수 코드 리뷰 (요구사항 모름) |
| `spec-reviewer` agent | 요구사항 리뷰 (요구사항 전달) |
| `code-quality` skill | frontend-doc 기반 코드 품질 평가 기준 |
| `spec-compliance` skill | 요구사항 충족도 평가 기준 |

       │ 80점 이상

       ▼

## Jira CLI 요구사항

- `jira-cli` 설치 필요 (macOS: `brew install ankitpokhrel/jira-cli/jira-cli`)
- `jira init`으로 로그인 설정 필요
- `/sunday:setup` 명령어로 안내에 따라 설정 가능
